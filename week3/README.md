# 1. 디자인 시스템 — Changeset 자동화, 그런데 아무도 못 쓴다

## 문제

디자이너가 개발자 도움 없이 디자인 토큰을 직접 Git에 반영할 수 있는 흐름을 만들고 싶었습니다.

기존 대안인 Token Studio는 워크플로우 자체는 좋았지만, 자동 Git Sync를 사용하려면 월 8만 원 수준의 유료 플랜이 필요했습니다.

무료 플랜도 가능은 했지만, 토큰을 단일 파일로 합쳐야 했습니다.

우리 팀은 아래처럼 토큰을 목적별로 분리해 관리하고 있습니다.

- color
- typography
- spacing

즉, 현재 운영 방식과 맞지 않았습니다.

---

## 시도

그래서 우회 방법으로 **AI Agent 기반 Changeset 자동화 스킬**을 만들었습니다.

디자이너가 단순히 아래처럼 요청하면 됩니다.

> changeset 만들어줘

그러면 아래 과정을 자동 수행합니다.

1. diff 분석 후 영향 패키지 탐지
2. 버전 bump 추천 (patch / minor / major)
3. 변경 유형 분류
4. 릴리즈 노트 문구 작성
5. changeset 생성 + 커밋

핵심은 **비개발자 친화성**이었습니다.

자유 입력 대신 선택지 중심으로 설계해 semver나 changeset 문법을 몰라도 사용할 수 있도록 구성했습니다.

---

## 결과

기술적으로는 꽤 만족스러웠습니다.

- 비개발자도 몇 번의 선택만으로 배포 준비 가능
- CHANGELOG 품질 균일화
- 버전 판단 자동화
- 커뮤니케이션 비용 감소 가능성 확인

---

## 현실

문제는 다른 곳에 있었습니다.

이 스킬은 Claude / Cursor 등 유료 플랜 사용자만 활용 가능했습니다. 현재 팀 디자이너들은 아직 해당 구독이 없는 상태였습니다.

즉,

> 자동화는 완성됐지만, 실제 사용자는 없었습니다.

결국 지금 시점에서는 개발자가 직접 changeset을 작성하는 방식이 더 효율적이었습니다.

---

## 남은 가치

다만 이번 작업은 버려지지 않습니다.

향후 아래 흐름으로 확장 가능합니다.

> 디자이너 토큰 수정 → PR 자동 생성 → 검토 → 머지

즉, 지금은 시기상조였지만 방향성은 유효했습니다.

# 2. 테크 블로그 — 품질은 높았지만, 기억되진 않았다

## 현황

4월 10일부터 운영 중인 테크 블로그는 현재 총 56편까지 작성되었습니다. 하루 평균 약 3편 수준입니다.

직접 보기엔 완성도가 높았지만 객관적 검증이 필요하다고 판단했습니다.

그래서 GPT / Gemini / Claude 계열 모델 4종에게 교차 평가를 요청했습니다.

---

## 공통 강점

네 모델이 대체로 동의한 강점은 아래와 같았습니다.

### 1) 기술 정확성

- 할루시네이션 거의 없음
- 참조 링크 유효
- 기술 설명 신뢰도 높음

### 2) 운영 완성도

- 카테고리 구조
- SEO 설계
- 시리즈 연결성
- 주제 선정 감각

즉,

> 단순히 글 몇 편 만든 수준이 아니라, 운영 시스템으로서 완성도가 높다는 평가였습니다.

---

## 공통 약점

반대로 약점도 명확했습니다.

### 1) 실전 경험 부족

- 실패 사례 없음
- 운영 중 문제 해결 경험 부족
- 왜 이 선택을 했는지 맥락 부족

즉 읽히긴 하지만 깊게 남지는 않는다는 평가였습니다.

### 2) AI 특유의 반복 패턴

- 비슷한 도입부
- 비슷한 문장 리듬
- 지나치게 정돈된 구조

몇 편 연속으로 읽으면 패턴이 보인다는 피드백도 있었습니다.

---

## 핵심 해석

이번 평가를 한 문장으로 요약하면 이렇습니다.

> 좋은 글 자동 생성 시스템은 만들었다.
> 하지만 아직 사람의 경험이 담긴 미디어는 아니다.

# 3. 다음 2주 계획 — 읽히는 블로그에서 기억되는 블로그로

문제는 비교적 선명합니다. 그래서 개선 방향도 명확합니다.

## 콘텐츠 측면

- 글 수를 줄이고 깊이를 늘리기
- 실제 사례 / 장애 경험 / 선택 이유 추가
- 본문 내 출처 링크 강화
- 문체 다양화

## 운영 측면

- 주제 자동 발굴
- 리서치 자동화
- 검수 자동화
- 발행 후 품질 모니터링

즉,

> 생산량 중심 시스템에서 신뢰도와 인상 중심 시스템으로 전환하려고 합니다.
